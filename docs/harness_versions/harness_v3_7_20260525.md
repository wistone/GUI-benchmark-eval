# Harness V3.7 Prompt + Dynamic Guardrails

Date: 2026-05-25

Purpose: reject V3.6 scale-up, preserve the V3.5/V3.6 download and Chrome wins, recover the V3.6 Impress regression, and reduce repeated command-entry/probing failures without adding per-task solutions.

Key changes from V3.6:
- Version marker and metadata updated to V3.7.
- The runner now injects dynamic Harness state warnings into each prompt, derived from recent action history.
- Repeated long text/script entry triggers an explicit guardrail: do not enter another long command; recover input state, run one short decisive command, verify, switch route, or FAIL.
- Repeated Terminal probes after mid-task trigger a guardrail to produce/verify the final artifact instead of continuing probes.
- Repetitive PDF/form and download-set tasks get task-type guardrails when the action history shows too much Terminal probing.
- Exact color tasks get a dynamic reminder to use named basic colors or explicit RGB/custom dialogs, not visually similar swatches.

Expected effect:
- Reduce max-step failures caused by repeated paste/type/probe loops.
- Preserve successful download and Chrome behavior.
- Recover exact Impress color behavior by avoiding approximate palette choices.

Risk:
- Guardrails are still prompt-level; they may not fix tasks that need a true VM-side script delivery primitive.
- If V3.7 still repeats commands or regresses preservation tasks, the next iteration should add a stricter runner-level intervention or reduce the action budget for repeated non-progress.
