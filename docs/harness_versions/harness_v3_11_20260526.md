# Harness V3.11 Color/PDF Guard

Date: 2026-05-26

Purpose: preserve V3.10's Writer/download/Chrome recovery while addressing the remaining small-sample failures.

Key changes from V3.10:
- Version marker and metadata updated to V3.11.
- Exact named Office colors now warn against inventing web/CSS RGB values when the task names an app color.
- Runner rejects pure `00FF00`/RGB `(0,255,0)` for exact named Office green unless the task explicitly provides that code, steering the model back to the application's Basic/Standard named palette.
- Repetitive PDF/form generation tasks now warn against interactive editor routes and reject early `FAIL` after script-entry attempts before one concrete noninteractive generation attempt and artifact verification.

Expected effect:
- Keep the V3.10 Office early-DONE fix for Writer.
- Restore Impress exact-color behavior without re-enabling broad Terminal/download rejection.
- Give PDF/form generation one decisive execution attempt instead of failing after a pasted-but-unexecuted script.

Risk:
- The named-color guard may still not force the model to the correct app palette value if the UI route is confusing.
- The PDF early-FAIL guard may spend additional steps on a weak script route; if it loops, the next version should move the intervention from prompt/rejection to a stricter noninteractive-script route.
