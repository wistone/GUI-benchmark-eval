# Harness V3.9 Narrow Intervention

Date: 2026-05-25

Purpose: preserve the useful part of V3.8 while removing the harmful part. V3.8 recovered exact Impress color and preserved Chrome, but broad Terminal/download action rejection failed the download task and regressed Writer.

Key changes from V3.8:
- Version marker and metadata updated to V3.9.
- Broad repeated Terminal/download/text-entry action rejection is disabled.
- Exact-color repeated right-palette coordinate click rejection is retained.
- Office tasks now reject early `DONE` after saving if no post-save verification action is visible.

Expected effect:
- Keep V3.8's exact-color recovery path.
- Restore V3.7/V3.6-style download behavior by not blocking repeated Terminal download commands.
- Reduce premature Office `DONE`, especially Writer line-spacing tasks that save and stop before checking the actual property.

Risk:
- Without broad Terminal rejection, command-heavy tasks may again burn step budget.
- The Office early-DONE gate may force extra verification steps but still not guarantee the right property is selected.
