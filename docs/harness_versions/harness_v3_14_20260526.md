# Harness V3.14 Download Folder Guard

Date: 2026-05-26

Purpose: keep V3.13's Writer/Impress/Chrome preservation while fixing the repeated numbered PDF download regression where the model downloaded into `~/Downloads` or `~` instead of the task's opened folder.

Key changes from V3.13:
- Version marker and metadata updated to V3.14.
- Kept V3.12 named-green guidance, V3.10 Office DONE gate, and V3.13 repeated download drift guard.
- Added a generic opened-folder hint extractor from VM setup actions such as `nautilus /home/user/...` and seeded download paths.
- For repeated numbered PDF download-set tasks, reject a final loop-and-verify command if it uses `~/Downloads` or bare `~` while a more specific opened/seeded target folder is available.
- Add a late warning that names the opened/seeded destination folder and tells the model to use that folder instead of default Downloads.

Expected effect:
- Preserve Writer, Impress, and Chrome.
- Recover download tasks that require placing a numbered PDF set into an already opened folder.

Risk:
- The destination guard depends on generic setup hints; tasks with intentionally different final destinations may need the model to mention the correct folder explicitly.
- PDF form generation remains unchanged because broad PDF intervention caused regressions in V3.11.
