# Harness V3.8 Runner-Level Intervention

Date: 2026-05-25

Purpose: reject V3.7 scale-up and move beyond prompt-only warnings. V3.7 repaired Writer line spacing, but failed exact Impress color, PDF generation, and even regressed the download preservation task through repeated Terminal command loops.

Key changes from V3.7:
- Version marker and metadata updated to V3.8.
- The runner now evaluates each proposed action before executing it.
- If a non-final action repeats Terminal/text-entry patterns beyond a generic threshold, the runner rejects it and calls the model again on the same screenshot with an explicit replan notice.
- Rejection applies to repeated Terminal command families such as download commands, Python scripts, list/verify probes, text extraction, and Office CLI commands.
- Rejection also applies to repeated long text/script entry after repeated long entries.
- Exact-color tasks now reject repeated right-side palette coordinate clicks, pushing the model toward named basic colors, explicit RGB/custom dialogs, or file-structure edits.

Expected effect:
- Stop repeated `wget`/`curl`/`ls`/`find` loops before they burn the step budget.
- Force earlier DONE/FAIL or a genuinely different route when a command path is not making progress.
- Preserve V3.7's Writer repair behavior.

Risk:
- Rejection may force earlier FAIL without improving correctness if the model cannot find a different route.
- Thresholds may be too aggressive for tasks where several Terminal commands are genuinely needed. Keep validation small before any 20-30 run.
