整体主线

1. 先让动作变干净

V1/V2 主要是清理 prompt、action schema，并探索 screenshot、a11y tree、OCR 等观测输入。

2. 防止过早 DONE

V3 开始强调每次动作后的状态检查，文件、设置、导出任务不能只看 UI 感觉。

3. 从原则到剧本

V3.1 到 V3.5 把“精确范围、不要近似替代、保存后验证”变成 Office、下载、VS Code 的具体规则。

4. 处理重复循环

V3.6 到 V3.10 重点处理长命令输入、Terminal 循环、早停 DONE，并引入 runner 层拦截。

5. 小步保守修复

V3.11 到 V3.17 以 preservation 为核心，发现过宽干预会回归后，逐步收窄 guard。
